## Подготовка

### 1. Apache JMeter 5.6+

Скачать: [https://jmeter.apache.org/download_jmeter.cgi](https://jmeter.apache.org/download_jmeter.cgi)  
Распаковать и задать переменную:

```bat
setx JMETER_HOME "C:\apache-jmeter-5.6.3"
```

### 2. Доступ к серверу

Сервер доступен **только из сети кафедры** или через **SSH-туннель** на helios:

```bat
scripts\tunnel.bat
```

В `tunnel.bat` замените `sXXXXXX` на свой логин.

Проверка:

```bat
curl http://localhost:8083/?token=492556914^&user=2109717879^&config=1
```

Ожидается HTTP **200** (не 403/503).

### 3. Python (для графика стресс-теста)

```bat
pip install matplotlib
```

## Структура JMeter-планов

### Нагрузочный тест (`load/load-test-plan.jmx`)

- **3 Thread Group** (config 1 → 2 → 3), выполняются **последовательно**
- В каждой группе:
  - **12 потоков**, ramp-up **60 с**
  - **40 итераций** на поток
  - **Constant Throughput Timer:** 20 запр./мин на поток (`calcMode=0`)
  - **Response Assertion:** HTTP 200 (не 403/503)
  - **SLA 640 мс** — проверяется при анализе CSV, **не** через Duration Assertion (иначе JMeter показывает ложный Error %)
- Listeners: Aggregate Report, Aggregate Graph, Response Time Graph

### Стресс-тест (`stress/stress-test-plan.jmx`)

- **1 Thread Group**, выбранная конфигурация (`config=2` по умолчанию)
- **48 потоков**, ramp-up **240 с**, длительность **300 с**
- 20 запр./мин на поток → нагрузка **растёт** по мере старта потоков
- После нагрузочного теста **измените `config`** на выбранную конфигурацию

## Анализ результатов

### Нагрузочный тест

Откройте `load_report/index.html` или Aggregate Report в GUI.

Для каждой конфигурации смотрите:

- **Max / 90% Line / Average** (elapsed) — должно быть ≤ 640 мс
- **Error %** — должно быть 0% (нет 503)
- **Throughput** — ≈ 4/sec (240/min)

**Выбор конфигурации:** самая **дешёвая** из прошедших SLA:


| Config | Цена   | Типичное поведение                     |
| ------ | ------ | -------------------------------------- |
| 1      | $4100  | Медленнее, может не уложиться в 640 мс |
| 2      | $7400  | Средняя                                |
| 3      | $10500 | Быстрее                                |


Заполните таблицу **своими** значениями из отчёта.

### Стресс-тест

1. Укажите выбранный `config` в `stress/stress-test-plan.jmx`
2. Запустите `run-stress.bat`
3. График: `stress/stress-graph.png`
4. **Точка отказа** — нагрузка (число потоков), при которой:
  - средний/макс. отклик **> 640 мс**, или
  - появляются ответы **503**

## Коды ответов


| Код | Значение                                   |
| --- | ------------------------------------------ |
| 200 | OK                                         |
| 403 | Некорректные параметры (token/user/config) |
| 503 | Сервер не справляется с нагрузкой          |
